# AI Software Engineer Agent - Parte 14

# User Interface e IDE Integration: costruire una esperienza tipo Cursor locale

## Obiettivo

Definire l'interfaccia con cui l'utente interagisce con l'agente.

L'obiettivo non è creare semplicemente una chat.

Una chat tradizionale è insufficiente per un software engineer agent perché manca:

* visione dello stato del progetto;
* controllo delle modifiche;
* approvazione;
* navigazione codice;
* feedback continuo.

L'interfaccia deve essere più vicina a:

```text id="7p2m9k"
ChatGPT

+

VS Code

+

Git Review

+

Project Dashboard
```

# 14. Sicurezza UI

L'agente deve avere permessi limitati.

Separare:

## Read permission

Sempre consentita.

---

## Write permission

Richiede:

* approvazione;
* oppure regola automatica.

---

## Execute permission

Controllata.

Esempio:

Consentito:

```bash
pytest
```

Richiede conferma:

```bash
rm -rf
docker compose down
```

# Principio finale

La UI non deve nascondere l'agente.

Deve renderlo osservabile.

Un buon AI Software Engineer non è quello che modifica più codice, ma quello che permette all'utente di capire:

* cosa sta facendo;
* perché lo fa;
* quale impatto ha;
* quando intervenire.
