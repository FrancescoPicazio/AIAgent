# AI Software Engineer Agent - Parte 36

# Advanced Coder Agent: generazione codice, modifica controllata, diff intelligente, self-debugging e collaborazione tra agenti

## Obiettivo

Definire l'agente responsabile della scrittura e modifica del codice.

Il Coder Agent non deve essere un semplice generatore di snippet.

Deve comportarsi come uno sviluppatore che lavora all'interno di un progetto esistente.

---

# 1. Input del Coder Agent

Il Coder non riceve solo:

> "Crea login"

Riceve un pacchetto completo.

---

Esempio:

```json
{
task:

"Implement JWT refresh token",


files:

[
"auth/service.py",
"auth/models.py"
],


constraints:

[
"Use existing database",
"Keep API compatibility"
],


tests:

[
"test_auth.py"
]

}
```

# 3. Code Modification Strategy

Il Coder non deve riscrivere file interi.

---

Strategia preferita:

```text
Locate

↓

Analyze

↓

Patch

↓

Validate

```

---

Esempio:

Modifica:

```python
def login():
```

---

Produce:

```diff
@@ login()

+ validate_refresh_token()

+ create_new_access_token()

```

---

# 4. Diff First Development

Ogni modifica passa dal diff.

---

Pipeline:

```text
Generate Change

↓

Create Diff

↓

Analyze Impact

↓

Human Approval

↓

Apply

```

---

Questo evita:

* modifiche accidentali;
* cancellazioni;
* regressioni.

---

# 5. Code Generation Rules

Il Coder deve rispettare regole globali.

File:

```text
.ai/

coding_rules.md

```

---

Esempio:

```markdown
# Coding Rules


- Clean Architecture

- Type hints obbligatori

- Ogni funzione documentata

- Nessuna funzione >50 linee

- Test obbligatori

```

---

# 6. Function Documentation

Come definito:

ogni funzione modificata deve avere:

* descrizione;
* parametri;
* output;
* errori.

---

Esempio:

```python
def calculate_price(cart):
    """
    Calculate total order price.

    Args:
        cart:
            List of products.

    Returns:
        float:
            Final price.

    Raises:
        ValueError:
            Invalid product.
    """
```

# Principio finale

Un buon Coder Agent non è quello che scrive più codice.

È quello che modifica meno codice possibile ottenendo il risultato corretto.

La qualità di un software engineer si misura dalla capacità di cambiare un sistema senza romperlo.
