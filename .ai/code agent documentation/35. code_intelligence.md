# AI Software Engineer Agent - Parte 35

# Code Intelligence Engine: AST, dependency graph, semantic search e comprensione profonda del repository

## Obiettivo

Definire il sistema che permette all'agente di comprendere un progetto software prima di modificarlo.

Un LLM che legge solo file di testo ha una visione limitata.

Un software engineer reale non apre file casualmente.

Prima costruisce un modello mentale:

* struttura del progetto;
* responsabilità dei moduli;
* dipendenze;
* flusso dei dati;
* punti critici.

Il Code Intelligence Engine crea questo modello.

---

# Principio fondamentale

Il codice non è solo testo.

È una struttura.

Esempio:

```python
def login(user):
    token = create_token(user)
    return token
```

Per un umano:

```text
login

↓

chiama create_token

↓

restituisce token
```

Per l'agente deve diventare:

```json
{
"type":"function",
"name":"login",
"calls":[
"create_token"
]
}
```

# 3. AST Parser

AST = Abstract Syntax Tree.

Permette di capire la struttura del codice.

---

Esempio:

Codice:

```python
class UserService:

    def create_user(self):
        pass
```

---

AST:

```json
{
class:
"UserService",

methods:[
"create_user"
]
}
```
