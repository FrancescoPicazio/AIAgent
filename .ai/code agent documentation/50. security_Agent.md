# AI Software Engineer Agent - Parte 50

# Security Agent: secure coding, vulnerability analysis, dependency scanning, secrets detection e policy enforcement

## Obiettivo

Definire l'agente responsabile della sicurezza del software durante tutto il ciclo di sviluppo.

Il Security Agent non interviene solo alla fine.

Deve essere presente in:

* progettazione;
* sviluppo;
* code review;
* testing;
* deployment;
* manutenzione.

---

# 1. Responsabilità Security Agent

Gestisce:

* analisi codice;
* vulnerabilità;
* dipendenze;
* segreti;
* configurazioni;
* compliance;
* sicurezza architetturale.

---

Non deve:

* modificare autonomamente codice critico senza approvazione;
* disabilitare controlli.

# 3. Static Application Security Testing (SAST)

Analizza il codice senza eseguirlo.

---

Cerca:

* SQL injection;
* command injection;
* path traversal;
* hardcoded secrets;
* problemi crittografici.

---

Esempio:

Codice:

```python id="v8m3q5"

query = "SELECT * FROM users WHERE id=" + user_id

```

---

Security Agent:

```text id="n7m2x8"

CRITICAL

SQL Injection risk

Use parameterized query

```

---

# 4. Dynamic Application Security Testing (DAST)

Analizza applicazione in esecuzione.

---

Esempio:

```text id="m4q8x1"

Start application

↓

Send malicious requests

↓

Analyze response

```

---

Trova:

* endpoint vulnerabili;
* configurazioni errate;
* errori esposti.
