# AI Software Engineer Agent - Parte 5

# Tool Layer: strumenti operativi degli agenti

## Obiettivo

Definire il livello degli strumenti che permette agli agenti AI di interagire realmente con il progetto software.

Il modello linguistico non deve avere accesso diretto e incontrollato al sistema.

Ogni operazione deve passare attraverso tool controllati e tracciabili.

Gli agenti ragionano.

I tool eseguono.

---

# Principio architetturale

Separazione tra:

```text id="8v0j4h"
LLM Agent

    |
    | decide cosa fare
    v

Tool Layer

    |
    | esegue operazioni
    v

Sistema operativo / Repository
```

L'agente non deve:

* manipolare file direttamente;
* eseguire comandi arbitrari;
* accedere a dati esterni senza permesso.

---

# 1. File Management Tools

## read_file()

## Scopo

Leggere il contenuto di un file.

Input:

```json
{
"path":
"src/auth/service.py"
}
```

Output:

```json
{
"path":
"src/auth/service.py",

"content":
"...",

"metadata":
{
"language":"python",
"lines":120
}
}
```

---

## write_file()

## Scopo

Creare un nuovo file.

Utilizzato per:

* nuovi moduli;
* configurazioni;
* documentazione.

Non deve sovrascrivere file esistenti senza autorizzazione.

---

## apply_patch()

## Scopo

Applicare modifiche atomiche.

Preferito rispetto a `write_file()`.

Input:

```json
{
"file":
"auth.py",

"patch":
"diff content"
}
```

Vantaggi:

* rollback;
* controllo;
* tracciabilità.

---

## delete_file()

Operazione critica.

Richiede:

```text
Human Approval
```

Sempre.

# 2. Code Search Tools

## search_code()

Ricerca testuale.

Esempio:

```text
search_code("JWT")
```

Restituisce:

```text
auth/service.py:42
middleware/auth.py:15
```

---

## semantic_search()

Ricerca tramite Vector Database.

Esempio:

Input:

```text
"Dove viene gestita l'autenticazione?"
```

Output:

```text
auth/service.py

UserController

JWT middleware
```

---

## find_symbol()

Ricerca strutturale tramite AST.

Esempio:

Input:

```text
UserService.login
```

Output:

```text
file:
user/service.py

line:
85
```

# 4. Execution Tools

## run_command()

Esegue comandi controllati.

Esempi:

```bash
python app.py

npm test

pytest
```

Non permette:

* accesso root;
* comandi fuori sandbox;
* operazioni distruttive.

# Principio finale

Gli agenti non controllano direttamente il computer.

Operano tramite strumenti controllati.

Questo permette:

* sicurezza;
* rollback;
* osservabilità;
* qualità;
* autonomia controllata.

Il Tool Layer è il confine tra il ragionamento dell'AI e il mondo reale del progetto.
