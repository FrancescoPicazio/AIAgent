# AI Software Engineer Agent - Parte 37

# Advanced Tester Agent: generazione test, execution loop, debugging automatico, coverage e quality gate

## Obiettivo

Definire l'agente responsabile della validazione del software.

Il Tester Agent non deve limitarsi a lanciare test esistenti.

Deve comportarsi come un QA Engineer senior:

* comprendere cosa deve funzionare;
* creare test mancanti;
* trovare errori;
* analizzare fallimenti;
* verificare regressioni;
* certificare la qualità prima dell'approvazione.

# Principio finale

Il Tester Agent non deve essere un controllore finale.

Deve essere un collaboratore continuo.

Il ciclo corretto non è:

```text
Scrivo → Verifico
```

ma:

```text
Creo → Misuro → Correggo → Miglioro
```

È questo ciclo che trasforma un generatore di codice in un vero processo software engineering.
