# AI Software Engineer Agent - Parte 39

# Advanced Research Agent: analisi documentazione, scelta tecnologie, confronto librerie e aggiornamento conoscenza del progetto

## Obiettivo

Definire l'agente incaricato della ricerca tecnica.

Il Research Agent serve quando il sistema incontra un problema dove il codice esistente non è sufficiente.

Esempi:

* scelta di una libreria;
* comprensione di una nuova tecnologia;
* analisi di API esterne;
* valutazione di alternative architetturali;
* aggiornamento rispetto a nuove versioni software.

---

# 1. Responsabilità principali

Il Research Agent deve:

* raccogliere informazioni;
* analizzare fonti;
* sintetizzare conoscenza;
* confrontare soluzioni;
* produrre raccomandazioni.

---

Non deve:

* decidere autonomamente senza vincoli;
* sostituire il Planner;
* modificare codice direttamente.

---

# Principio finale

Un software engineer esperto non conosce tutto.

Sa soprattutto:

* cosa deve cercare;
* dove cercarlo;
* come valutare le alternative;
* come trasformare informazioni in decisioni tecniche.

Il Research Agent porta questa capacità all'interno dell'AI Software Engineer.
