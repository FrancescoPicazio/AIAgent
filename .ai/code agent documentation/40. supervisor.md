# AI Software Engineer Agent - Parte 40

# Supervisor Agent: orchestrazione completa, routing degli agenti, gestione stato e ciclo decisionale globale

## Obiettivo

Definire il componente che coordina tutti gli agenti del sistema.

Il Supervisor è il "sistema nervoso centrale" dell'AI Software Engineer.

Non scrive codice.

Non esegue test.

Non fa ricerca.

Decide:

* quale agente deve intervenire;
* quando intervenire;
* quali informazioni passare;
* quando fermarsi;
* quando chiedere approvazione umana.

---

# Principio fondamentale

Un sistema multi-agent senza Supervisor:

```text id="m7q3x8"

User

 |

Coder

 |

Tester

 |

Security


```

Problema:

* conflitti;
* perdita contesto;
* loop;
* decisioni incoerenti.

---

Sistema con Supervisor:

```text id="q8n4m2"


                  User


                   |


             Supervisor


                   |


 +---------+-------+-------+---------+

 |         |       |       |         |

Planner  Coder  Tester Security Research


                   |

              Memory Layer


```

---

# Responsabilità Supervisor

## 1. Interpretazione richiesta

Capisce cosa vuole l'utente.

---

## 2. Routing

Decide quale agente chiamare.

---

## 3. Stato globale

Mantiene memoria del lavoro corrente.

---

## 4. Controllo qualità

Verifica che il ciclo sia completato.

---

## 5. Human Approval

Gestisce i punti dove serve conferma.
