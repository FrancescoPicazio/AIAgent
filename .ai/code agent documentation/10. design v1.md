2. creare modello User;

# Cosa NON implementare nella v1

Evitare:

* Vector DB;
* AST completo;
* Graph DB;
* multi-language;
* auto commit;
* self learning.

Motivo:

Prima bisogna validare il ciclo base.

* memoria avanzata;
* comprensione codice;
* analisi impatto.

La v3 aggiungerГ :

* autonomia;
* ottimizzazione;
* capacitГ  da senior engineer.
