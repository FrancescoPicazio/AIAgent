### 2. Semantic Code Analysis

#### 2.1 Code Semantics Extraction

```python
class CodeSemantics:
    """Semantic meaning of code."""
    
    def __init__(self, entity_id: str, entity_type: str):
        self.id = entity_id
        self.type = entity_type          # "function", "class", "module"
        self.purpose: str = ""           # What does it do?
        self.responsibilities: List[str] = []  # Main responsibilities
        self.interfaces: List[str] = []  # Public interface
        self.complexity: int = 0         # Cyclomatic complexity
        self.maintainability: float = 0.0  # 0-100 score
```

#### 2.2 Semantic Analysis Process

```python
def analyze_semantics(self, code: str, entity_id: str) -> CodeSemantics:
    """Analyze semantic meaning of code."""
    
    semantics = CodeSemantics(entity_id, "function")
    
    # 1. Extract purpose from docstring
    purpose = self._extract_purpose(code)
    
    # 2. Identify responsibilities from implementation
    responsibilities = self._identify_responsibilities(code)
    
    # 3. Calculate complexity metrics
    complexity = self._calculate_cyclomatic_complexity(code)
    maintainability = self._calculate_maintainability_index(code)
    
    # 4. Identify interfaces
    interfaces = self._extract_public_interface(code)
    
    # 5. Generate semantic embedding
    embedding = self._create_semantic_embedding(code, purpose, responsibilities)
    
    semantics.purpose = purpose
    semantics.responsibilities = responsibilities
    semantics.complexity = complexity
    semantics.maintainability = maintainability
    semantics.interfaces = interfaces
    
    return semantics
```

### 3. Impact Analysis

#### 3.1 Change Impact Assessment

```python
class ImpactAssessment:
    """Assessment of code change impact."""
    
    def __init__(self, change_id: str):
        self.change_id = change_id
        self.directly_affected: Set[str] = set()      # Direct dependencies
        self.indirectly_affected: Set[str] = set()    # Transitive dependencies
        self.risk_level: str = "LOW"                  # LOW/MEDIUM/HIGH/CRITICAL
        self.affected_areas: Dict[str, str] = {}      # Area -> impact description
        self.cascade_risks: List[str] = []            # Potential cascade failures
```

#### 3.2 Impact Calculation

```python
def calculate_impact(self, changed_entity: str, graph: DependencyGraph) -> ImpactAssessment:
    """Calculate impact of changing an entity."""
    
    assessment = ImpactAssessment(changed_entity)
    
    # 1. Find direct dependents
    direct = self._find_direct_dependents(changed_entity, graph)
    assessment.directly_affected = direct
    
    # 2. Find transitive dependents
    transitive = self._find_transitive_dependents(changed_entity, direct, graph)
    assessment.indirectly_affected = transitive
    
    # 3. Calculate risk level
    total_affected = len(direct) + len(transitive)
    if total_affected == 0:
        assessment.risk_level = "LOW"
    elif total_affected <= 5:
        assessment.risk_level = "MEDIUM"
    elif total_affected <= 20:
        assessment.risk_level = "HIGH"
    else:
        assessment.risk_level = "CRITICAL"
    
    # 4. Identify affected areas
    for affected_entity in direct.union(transitive):
        area = self._identify_affected_area(affected_entity)
        impact = self._describe_impact(changed_entity, affected_entity)
        assessment.affected_areas[area] = impact
    
    # 5. Identify cascade risks
    assessment.cascade_risks = self._identify_cascade_risks(changed_entity, graph)
    
    return assessment
```

**Example:**
```
Changed: calculate_price()

Direct dependents:
  - apply_discount()
  - generate_invoice()
  - validate_order()

Transitive dependents:
  - process_payment()
  - send_notification()
  - update_inventory()

Risk Level: HIGH
Affected Areas: {
  "payment": "Invoice calculation affected",
  "notifications": "Receipt generation may fail",
  "inventory": "Stock updates may be incorrect"
}
```

class SemanticSearchQuery:

#### 4.2 Search Types

   - Returns: All validation functions, ranked by relevance

2. **Dependency Search**

3. **Pattern Search**

4. **Risk Search**

### 5. Refactoring Suggestions

#### 5.1 Refactoring Opportunities

```python
class RefactoringSuggestion:
    """Suggestion for code refactoring."""
    
    def __init__(self, suggestion_type: str):
        self.type = suggestion_type     # "extract_method", "consolidate", etc
        self.location: str = ""         # File and line number
        self.current: str = ""          # Current code
        self.suggested: str = ""        # Suggested code
        self.rationale: str = ""        # Why refactor?
        self.impact: str = ""           # Expected impact
        self.effort: str = ""           # LOW/MEDIUM/HIGH
```

#### 5.2 Refactoring Types

- **Extract Method**: Long functions > 50 lines
- **Consolidate Conditionals**: Complex if statements
- **Remove Dead Code**: Unreachable code paths
- **Reduce Duplication**: Duplicate logic
- **Simplify Expressions**: Overcomplicated logic
