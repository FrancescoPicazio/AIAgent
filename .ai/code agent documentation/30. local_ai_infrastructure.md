# Local AI Infrastructure e Hardware Optimization: Ollama, RX 6800 XT, modelli locali e gestione delle risorse

## Obiettivo

Definire l'infrastruttura hardware e software necessaria per eseguire l'intero AI Software Engineer localmente.

Requisiti:

* nessun dato deve uscire dalla macchina;
* utilizzo di Ollama come runtime LLM;
* gestione di più modelli;
* ottimizzazione GPU;
* controllo VRAM/RAM;
* esecuzione stabile su hardware consumer.

---

# Principio fondamentale

Un agente complesso non deve dipendere da un unico modello.

Approccio errato:

```text id="m7q3x8"

↓

Tutti i task

```

---

Approccio corretto:

```text id="q8n4m2"

Task

↓

Model Router

↓

Modello specializzato

```

---

Esempio:

```text id="x5m9p1"

↓

Qwen3-Coder


Reasoning

↓

Gemma


Fast tasks

↓

Small model

```

# 3. Modelli consigliati

## Coding Model

Alibaba Cloud Qwen3-Coder

Utilizzo:

* generazione codice;
* refactoring;
* debugging;
* analisi repository.

---

Configurazione possibile:

```text id="m7q2n9"

Qwen3-Coder 30B

quantizzato

```

---

Problema:

30B è pesante per 16GB VRAM.

Necessario:

* quantizzazione;
* offload RAM.

---

# Reasoning Model

Google DeepMind Gemma

Utilizzo:

* analisi architetturale;
* pianificazione;
* decisioni.

---

# Lightweight Model

Qwen3 4B / modelli piccoli.

Utilizzo:

* classificazione;
* routing;
* task semplici.

---

# 4. Quantizzazione

Fondamentale su GPU consumer.

Riduce:

* VRAM;
* tempi caricamento.

---

Esempio:

FP16:

```text id="v5n8m2"

30B × 2 byte

≈60GB

```

---

Q4:

```text id="q8m3x1"

30B × 0.5 byte

≈15GB

```

---

Per RX 6800 XT:

target:

```text id="m6q3p9"

Q4/Q5 quantization
```

---
