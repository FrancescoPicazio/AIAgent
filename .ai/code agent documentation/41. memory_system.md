# AI Software Engineer Agent - Parte 41

# Memory System: .ai folder, Vector Database, RAG locale, Glossary, ADR e memoria evolutiva del progetto

## Obiettivo

Definire il sistema di memoria che permette all'agente di mantenere conoscenza del progetto nel tempo.

Senza memoria ogni conversazione è indipendente.

Il comportamento sarebbe:

```text
Giorno 1:

"Creo sistema login"


Giorno 2:

"Modifica login"


↓

L'agente non ricorda nulla

```

---

Con memoria:

```text
Giorno 1:

Analizza progetto

↓

Salva conoscenza


Giorno 2:

Recupera contesto

↓

Continua lavoro

```

---

# Principio fondamentale

La memoria non deve essere un semplice archivio.

Deve essere una rappresentazione strutturata della conoscenza del progetto.

---

# 2. Tipologie di memoria

Il sistema usa tre livelli.

---

# Short Term Memory

Memoria della sessione corrente.

Contiene:

* task corrente;
* errori;
* modifiche temporanee;
* conversazione.

Durata:

ore/giorni.

---

Esempio:

```json
{
task:

"Implement login",


current_file:

"auth.py"

}

```

---

# Working Memory

Memoria dello sviluppo corrente.

Contiene:

* sprint;
* piano;
* modifiche;
* problemi aperti.

---

File:

```text
.ai/state.json
```

---

# Long Term Memory

Conoscenza permanente.

Contiene:

* architettura;
* decisioni;
* pattern;
* regole.
