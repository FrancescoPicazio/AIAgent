## Obiettivo

Definire il cuore operativo dell'intero sistema.

Finora abbiamo creato diversi componenti:

* Product Owner Agent;
* Architect Agent;
* Planner;

---

# Principio fondamentale

Un sistema multi-agent non deve essere una semplice sequenza.

Approccio errato:

```text id="m7q3x8"

User

↓

Coder

↓

Tester

↓

Fine

```

---

Approccio corretto:

```text id="q8n4m2"

User

↓

Supervisor

↓

Decisione

↓

Agente specializzato

↓

Valutazione

↓

Nuova decisione

```

---

Il sistema deve poter decidere:

* quale agente chiamare;
* in quale ordine;
* quando fermarsi;
* quando chiedere approvazione.

# 2. Global State

LangGraph lavora tramite uno stato condiviso.

Esempio:

```python id="p4n8m2"
class AgentState:

    user_request: str

    project_context: dict

    current_task: str

    plan: list

    modified_files: list

    test_results: dict

    security_report: dict

    approval_required: bool

```

---

Ogni nodo legge e aggiorna lo stato.

# Principio finale

Il valore di un sistema multi-agent non è avere tanti agenti.

È avere un cervello che sa:

* quale agente usare;
* quando usarlo;
* cosa chiedergli;
* quando fidarsi;
* quando fermarsi.

L'Agent Runtime è il sistema nervoso dell'AI Software Engineer.
