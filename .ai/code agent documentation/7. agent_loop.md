# AI Software Engineer Agent - Parte 7

# End-to-End Development Workflow: dalla richiesta al commit

## Obiettivo

Definire il ciclo completo con cui l'agente AI trasforma una richiesta in linguaggio naturale in una modifica software verificata e integrata nel progetto.

Il workflow deve simulare un processo professionale di sviluppo software:

* analisi requisiti;
* progettazione;
* pianificazione;
* implementazione;
* verifica;
* revisione;
* aggiornamento conoscenza;
* versionamento.

---

# Visione generale del workflow

```text id="2zq0fh"
User Request

      ↓

Requirement Analysis

      ↓

Architect Agent

      ↓

Planner Agent

      ↓

Impact Analysis

      ↓

Human Approval

      ↓

Task Execution Loop

      ↓

Code Review

      ↓

Testing Pipeline

      ↓

Memory Update

      ↓

      ↓

Completion Report
```

# 1. User Request

## Input

L'utente comunica una modifica tramite chat.

Esempio:

```text id="r7w6j5"
"Aggiungi autenticazione tramite Google OAuth"
```

La richiesta viene salvata.

Percorso:

```text
.ai/tasks/active/task_x/request.md
```

---

# 2. Requirement Analysis

## Obiettivo

Trasformare una richiesta vaga in un requisito tecnico.

L'agente analizza:

* intenzione utente;
* vincoli;
* funzionalità richieste;
* possibili ambiguità.

---

Output:

```markdown id="9j8v4x"
# Requirement

Feature:

Google OAuth Authentication


Goal:

Permettere agli utenti di autenticarsi tramite account Google.


Expected behavior:

- Login Google
- Creazione utente se non esistente
- Generazione sessione autenticata


Constraints:

- mantenere JWT esistente
- non modificare API pubbliche
```

# 4. Planning Phase

Il Planner Agent crea task atomici.

Regola:

Ogni task deve essere eseguibile indipendentemente.

---

Esempio:

```yaml id="h1v8y9"
tasks:

- id: TASK-001

  title:
  Create OAuth interface

  files:
  - auth/interface.py


- id: TASK-002

  title:
  Implement Google provider

  files:
  - auth/google.py


- id: TASK-003

  title:
  Add authentication tests

  files:
  - tests/test_oauth.py
```

# Principio finale

L'agente non esegue una singola generazione di codice.

Segue un ciclo continuo:

```text
Capire

↓

Progettare

↓

Pianificare

↓

Modificare

↓

Verificare

↓

Imparare
```

Ogni modifica migliora la conoscenza futura del progetto.

L'obiettivo è creare un sistema che dopo settimane o mesi di lavoro conosca il progetto quasi come un membro umano del team.
