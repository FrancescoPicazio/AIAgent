# AI Software Engineer Agent - Parte 23

# Product Owner Agent: gestione requisiti, roadmap, user story e allineamento prodotto

## Obiettivo

Definire il livello che trasforma una richiesta generica in un piano di sviluppo concreto.

Un software engineer umano raramente riceve richieste perfette.

Normalmente riceve frasi come:

```text id="m8q4p2"
"Vorrei aggiungere una chat"

"Il sistema deve essere più veloce"

"Serve una dashboard"

"Facciamo una versione mobile"
```

---

Il compito del Product Owner Agent è trasformare queste idee in:

* requisiti chiari;
* priorità;
* user story;
* criteri di accettazione;
* roadmap;
* task tecnici.

---

# 1. Product Owner Agent

## Responsabilità

Gestisce:

* cosa costruire;
* perché costruirlo;
* quale valore porta;
* quale priorità ha.

Non decide:

* come implementare tecnicamente;
* quale framework usare.

Quello rimane dell'Architect Agent.

---

# 2. Requirement Understanding

L'agente analizza richieste incomplete.

---

Input:

```text id="p4m8x2"
"Voglio notifiche push"
```

---

Domande generate:

```yaml id="n7m3q9"
questions:

- per quali eventi?
- quali utenti?
- realtime?
- mobile o web?
- preferenze utente?
- storico notifiche?
```

# 3. Requirement Refinement Loop

Prima di creare task:

```text id="m6q2v8"

↓

↓

↓
```

---

---

Possibili interpretazioni:


---

Serve conferma.

# 4. User Story Generation

Trasforma requisiti in formato prodotto.

Template:

```text id="q3m8n2"

Come [tipo utente]

voglio [azione]

per ottenere [valore]
```

---

Esempio:

```markdown id="x5n7m1"
## User Story

Come utente registrato

voglio accedere tramite Google

per non dover creare una nuova password.


Priority:

High
```

---

# 5. Acceptance Criteria

Una feature non è completa se non ha criteri verificabili.

---

Formato:

```gherkin id="k8m3q6"
Given:

User has account


When:

User clicks Google login


Then:

Authentication succeeds
```

---

Utilità:

Diventa input per:

* Planner;
* Tester;
* Quality Agent.

---

# 6. Feature Breakdown

Il Product Owner divide una feature grande.

---

Esempio:

```text id="r4m8x1"

```

---

Diventa:

```text id="m7q2n9"

Stories:

2.
Create checkout flow


3.
Handle failures


4.
Add receipts


5.
Add refunds

```

# 7. Prioritization Engine

Non tutto deve essere sviluppato.

L'agente valuta:

---

## Value

Quanto valore porta?

---

## Effort

Quanto costa?

---

## Risk

Quanto è rischioso?

---

## Dependency

Blocca altro lavoro?

---

Formula esempio:

```text id="p9m3x5"

Priority Score =

(Value × Impact)

/

(Effort × Risk)

```

---

Output:

```yaml id="v6q8m2"
feature:

Push Notification


score:

8.7


priority:

P1
```

# Principio finale

Un AI Software Engineer completo non deve solo sapere:

"Come costruire qualcosa?"

Deve sapere anche:

"Perché costruirlo?"

Il Product Owner Agent chiude il divario tra idea e implementazione.

Senza questo livello l'agente rimane un ottimo programmatore.

Con questo livello diventa un vero partner nello sviluppo di prodotto.
