# AI Software Engineer Agent - Parte 11

# MVP v2: Code Intelligence Layer con AST, Vector Database e Dependency Graph

## Obiettivo

Evolvere l'agente dalla versione MVP, dove modifica codice seguendo il contesto fornito, verso un sistema capace di comprendere autonomamente un progetto software complesso.

La differenza principale:

MVP v1:

```text id="5xkq3v"
User

↓

Task

↓

Modifica codice
```

MVP v2:

```text id="p9x3ks"
User

↓

Comprensione progetto

↓

Analisi impatto

↓

Pianificazione intelligente

↓

Modifica codice
```

---

# Problema da risolvere

Nei progetti grandi il limite principale dei modelli linguistici è il contesto.

Un repository reale può contenere:

* migliaia di file;
* milioni di righe;
* centinaia di dipendenze.

Non è possibile inviare tutto il codice al modello.

Serve una memoria intelligente.


# Limiti MVP v2

Ancora non implementato:

* refactoring automatico globale;
* apprendimento dalle modifiche;
* gestione multi repository;
* deploy automatico;
* agenti paralleli.

---

# Milestone MVP v2 completata quando:

```text id="d8q5m3"
✓ Indicizza repository

✓ Comprende funzioni/classi

✓ Crea dependency graph

✓ Usa vector search

✓ Recupera contesto corretto

✓ Fa impact analysis

✓ Modifica codice con maggiore precisione
```

---

# Principio finale

MVP v1 dava all'agente mani.

MVP v2 gli dà memoria e comprensione.

La differenza tra un chatbot che scrive codice e un vero AI Software Engineer è la capacità di capire il sistema prima di modificarlo.
