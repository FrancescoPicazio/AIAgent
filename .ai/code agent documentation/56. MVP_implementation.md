# AI Software Engineer Agent - Parte 56

# Implementazione MVP reale: ordine di sviluppo, priorità tecniche e roadmap verso una prima versione funzionante

## Obiettivo

Definire l'ordine corretto di implementazione dell'AI Software Engineer.

L'errore più comune in sistemi complessi è cercare di costruire subito l'architettura completa.

L'obiettivo dell'MVP è ottenere rapidamente un sistema che:

* riceve una richiesta;
* comprende il progetto;
* modifica codice;
* esegue test;
* corregge errori;
* mantiene memoria.

---

# Principio fondamentale

Non costruire:

```text id="m7q3x8"

10 agenti perfetti

↓

Sistema completo

```

---

Costruire:

```text id="q8n4m2"

Un agente funzionante

↓

Aggiungere capacità

↓

Aumentare autonomia

```

---

## Step 3

Creare Project Analyzer.

Responsabilità:

* linguaggi;
* struttura;

---

## Step 4

Creare Coder Agent.

Funzioni:

* leggere file;
* proporre modifica;
* generare diff;
* applicare patch.

---

Importante:

non modificare direttamente.

Prima:

```text id="m4q8x1"

↓

Review

↓

Apply

```

---

## Step 5

Creare Tester Agent.

Esegue:

```bash

```

```bash

```

* stack trace;
* log.
