# AI Software Engineer Agent - Parte 16

# Long-Term Memory Layer: gestione contesto, conoscenza persistente e apprendimento del progetto

## Obiettivo

Definire il sistema di memoria che permette all'agente di lavorare sullo stesso progetto per settimane o mesi mantenendo conoscenza dello storico.

Il problema principale non è solo "ricordare".

È ricordare **le informazioni giuste al momento giusto**.

---

# Problema della memoria nei LLM

Un modello linguistico non possiede memoria permanente.

Ogni chiamata è:

```text id="g5m8q2"
Prompt

+

Context

↓

LLM

↓

Response
```

Se il contesto non viene fornito, il modello non sa:

* cosa è stato fatto ieri;
* perché una scelta architetturale è stata presa;
* quali errori sono già stati risolti;
* quali regole deve rispettare.

---

# Errore comune

Approccio sbagliato:

```text id="p8x3m1"
Mandare tutto il repository al modello
```

Problemi:

* token enormi;
* costi computazionali;
* rumore;
* perdita informazioni importanti.

---

# Soluzione

Creare una memoria a livelli.

Architettura:

```text id="m4q7n9"

                 Project

                    |

        +-----------+-----------+

        |           |           |

   Short Memory  Long Memory  Knowledge

        |           |           |

   Conversation  Vector DB   Graph DB

                    |

              Markdown Memory

```

---

# Memory Architecture

La memoria viene divisa in 5 livelli.

---

# Level 0 - Working Memory

## Obiettivo

Contesto temporaneo della task corrente.

Durata:

Solo durante esecuzione.

Contiene:

```json id="z7m3p4"
{
"current_task":
"Add authentication",

"files":
[
"auth.py"
],

"errors":
[]
}
```

---

Implementazione:

LangGraph State.

## Obiettivo

# Level 2 - Project Documentation Memory

## Obiettivo

```text id="q3m8p6"

├── project.md

├── architecture.md

├── rules.md

├── glossary.md

└── decisions.md
```

---

Contiene:

* descrizione progetto;

```markdown id="h6p2m9"
Backend:

FastAPI

PostgreSQL

Repository Pattern
```

---

## architecture.md

Contiene:

Decisioni strutturali.

Esempio:

# Principio finale

La memoria non serve a far ricordare tutto all'agente.

Serve a permettergli di dimenticare ciò che è inutile e recuperare solo ciò che serve.

Un vero software engineer non conosce ogni riga di codice a memoria.

Conosce:

* dove cercare;
* perché una scelta è stata fatta;
* quali conseguenze può avere una modifica.

La memoria artificiale deve replicare questo comportamento.
