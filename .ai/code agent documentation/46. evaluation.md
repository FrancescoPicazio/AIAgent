# AI Software Engineer Agent - Parte 46

# Evaluation System con LangSmith: benchmark, metriche, feedback loop e miglioramento continuo dell'agente

## Obiettivo

Definire il sistema che misura la qualità dell'intero AI Software Engineer.

Un agente che scrive codice ma non viene valutato non può migliorare.

L'obiettivo è trasformare il sistema da:

```text id="m7q3x8"

Input

↓

LLM

↓

Output

```

a:

```text id="q8n4m2"

Input

↓

Agent Workflow

↓

Evaluation

↓

Feedback

↓

Improvement

```

---

# Principio fondamentale

Il successo non è:

> "Il codice compila"

Il successo è:

* il requisito è stato soddisfatto;
* il codice è mantenibile;
* i test sono corretti;
* l'architettura è coerente;
* il comportamento è prevedibile.

# 1. Ruolo di LangSmith

LangSmith diventa il sistema di osservabilità e valutazione.

Registra:

* chiamate LLM;
* tool utilizzati;
* decisioni degli agenti;
* tempi;
* errori;
* risultati.

Security
