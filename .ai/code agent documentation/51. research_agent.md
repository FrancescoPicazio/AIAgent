# AI Software Engineer Agent - Parte 51

# Research Agent: analisi tecnologie, scelta librerie, confronto soluzioni, documentazione tecnica e decisioni architetturali

## Obiettivo

Definire l'agente responsabile della ricerca tecnica e del supporto decisionale.

Il Research Agent permette all'AI Software Engineer di non limitarsi a scrivere codice, ma di scegliere consapevolmente:

* framework;
* librerie;
* architetture;
* database;
* strumenti;
* pattern progettuali.

---

# 1. Responsabilità Research Agent

Gestisce:

* ricerca tecnica;
* confronto tecnologie;
* analisi documentazione;
* valutazione librerie;
* ADR (Architecture Decision Record);
* aggiornamento Knowledge Base.

---

# 2. Quando viene attivato

Non deve fare ricerca sempre.

Viene chiamato quando:

* manca conoscenza;
* esiste un'incertezza;
* una scelta ha impatto architetturale.

---

Esempi:

```text id="p4n8m2"

"Quale database usare?"

"Quale framework frontend scegliere?"

"Questa libreria è mantenuta?"

"Esiste una soluzione migliore?"

```

# 6. Technology Comparison Engine

Confronta alternative.

---

Esempio:

Richiesta:

> Scegli database per progetto

---

Analisi:

| Tecnologia | Pro                 | Contro           |
| ---------- | ------------------- | ---------------- |
| PostgreSQL | Relazionale, maturo | Più complesso    |
| MongoDB    | Flessibile          | Meno consistenza |
| SQLite     | Semplice            | Poco scalabile   |

---

Output:

```text id="x8m2q5"

PostgreSQL


Reason:

Need transactional consistency

```

# Principio finale

Un AI Software Engineer non deve solo sapere "come fare".

Deve sapere anche:

* perché scegliere una soluzione;
* quali alternative esistono;
* quali compromessi introduce.

La qualità del software nasce dalle decisioni, non solo dall'implementazione.
