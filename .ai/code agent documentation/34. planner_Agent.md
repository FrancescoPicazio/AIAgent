# AI Software Engineer Agent - Parte 34

# Advanced Planner Agent: decomposizione task, gestione dipendenze, stima complessità e coordinamento di grandi progetti

## Obiettivo

Definire il componente che trasforma una richiesta umana ad alto livello in un piano tecnico eseguibile.

Il Planner è uno degli agenti più importanti dell'intero sistema.

Un modello che sa scrivere codice ma non sa pianificare produce:

* codice incoerente;
* modifiche incomplete;
* architetture fragili;
* regressioni.

---

# Principio fondamentale

Il Planner non deve rispondere:

> "Come scrivo questo codice?"

Deve prima rispondere:

> "Qual è il lavoro necessario per arrivare al risultato?"

# 1. Requirement Analyzer

Prima del piano tecnico serve capire la richiesta.

---

Esempio:

Utente:

> "Voglio un sistema di autenticazione"

---

Il Planner deve estrarre:

```yaml id="q8n4m2"

feature:

authentication


questions:

- user types?

- login method?

- database?

- security requirements?


unknowns:

true

```

---

Non deve partire subito a scrivere.

---

# 2. Requirement Clarification Loop

Se mancano informazioni:

l'agente deve chiedere.

---

Esempio:

Utente:

> "Aggiungi pagamento"

---

Domande:

```text id="x5m9p1"

Quale provider?

Stripe?

PayPal?

Sistema interno?


Quali valute?

Serve rimborso?

```

---

Regola:

Meglio una domanda prima che 100 modifiche dopo.

# Principio finale

La differenza tra un generatore di codice e un software engineer è la pianificazione.

Un generatore dice:

> "Ecco il codice."

Un engineer dice:

> "Prima definiamo cosa serve, quali rischi abbiamo e qual è il percorso migliore per arrivare al risultato."

Il Planner è il componente che porta l'agente da autocomplete intelligente a vero collaboratore tecnico.
