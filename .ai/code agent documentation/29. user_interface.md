# AI Software Engineer Agent - Parte 29

# User Interface e Developer Experience: chat IDE, controllo progetto e interazione uomo-agente

## Obiettivo

Definire il livello con cui lo sviluppatore interagisce con l'AI Software Engineer.

L'obiettivo non è creare una semplice chat.

Deve essere un ambiente dove l'utente può:

* descrivere cosa vuole;
* vedere cosa sta facendo l'agente;
* approvare modifiche;
* controllare decisioni;
* navigare il progetto;
* comprendere lo stato del lavoro.

---

# Principio fondamentale

Una chat tradizionale è insufficiente.

Modello semplice:

```text id="m7q3x8"

User

↓

Chat

↓

AI risposta

```

---

Un AI Software Engineer necessita:

```text id="q8n4m2"

↓

↓

AI Agent

↓

Actions

↓

Review

↓

Approval

```

# Principio finale

L'interfaccia non deve nascondere l'agente.

Deve renderlo osservabile.

Un buon AI Software Engineer deve dare la sensazione:

"Sto lavorando con un collega molto veloce"

non:

"Sto parlando con una scatola nera".
