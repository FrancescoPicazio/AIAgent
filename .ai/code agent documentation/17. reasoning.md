# AI Software Engineer Agent - Parte 17

# Reasoning Layer: decision making, architettura, ADR e coerenza progettuale

## Obiettivo

Definire il livello che permette all'agente di prendere decisioni tecniche ragionate.

Un software engineer non si limita a scrivere codice.

Prima di modificare un sistema deve rispondere a domande come:

* questa è la soluzione migliore?
* esiste già un componente simile?
* quali sono i rischi?
* quale sarà l'impatto futuro?
* perché questa scelta è stata fatta?

Questo livello gestisce il "perché" dietro ogni modifica.

---

# Problema dei Coding Agent

Un agente puramente generativo tende a:

```text id="m3x7q9"
Richiesta

↓

Scrivi codice

↓

Fine
```

Problemi:

* crea duplicazioni;
* introduce pattern incoerenti;
* modifica senza considerare il futuro;
* risolve il sintomo invece della causa.

# 1. Understanding Agent

## Obiettivo

Capire cosa vuole realmente l'utente.

Una richiesta raramente è completa.

---

Esempio:

Utente:

```text id="p7m3x8"
"Rendi il sistema più veloce"
```

Non è un task.

È un obiettivo.

---

L'agente deve trasformarlo:

```yaml id="n4q8m2"
goal:

Improve performance


unknowns:

- backend?
- database?
- frontend?


required_analysis:

- profiling
- bottleneck detection
```

---

# 2. Problem Decomposition

Prima di scegliere una soluzione:

dividere il problema.

Esempio:

```text id="v5m8q1"
Slow API

|

+ Database query

+ Serialization

+ Network latency

+ Cache strategy
```

---

L'agente evita di modificare codice troppo presto.

---

# 3. Alternative Generation

Per ogni problema deve proporre più soluzioni.

Esempio:

Problema:

"Ridurre tempo risposta API"

---

Alternative:

```yaml id="x9m2p6"

solution_A:

Add Redis cache


pros:

- fast
- simple


cons:

- cache invalidation


solution_B:

Optimize database


pros:

- cleaner


cons:

- requires analysis
```

---

# 4. Decision Matrix

L'agente valuta le alternative.

Parametri:

| Criterio       | Peso |
| -------------- | ---- |
| Complessità    | 20%  |
| Manutenibilità | 30%  |
| Performance    | 25%  |
| Rischio        | 25%  |

---

Esempio:

```json id="k7m4p9"
{
"redis_cache":

score:
8.2,


"database_optimization":

score:
9.1
}
```

---

Scelta:

```text id="r3n8m5"
Optimize database first.
```

# 5. Architecture Decision Record (ADR)

Ogni decisione importante viene salvata.

Struttura:

```text id="h6q2m8"
.ai/

decisions/

ADR-001.md

ADR-002.md
```

---

Esempio:

```markdown id="c8m5q1"
# ADR-001

Title:

Use Repository Pattern


Date:

2026-07-11


Context:

Direct database access created coupling.


Decision:

All database operations must pass through repositories.


Consequences:

+ Better testing

+ Lower coupling


Trade-off:

More files
```

---

# 6. Decision Memory

L'ADR diventa memoria permanente.

Quando in futuro:

Utente:

```text id="w5m9x2"
"Crea nuovo modulo utenti"
```

---

L'agente recupera:

```text id="n3q7m4"
ADR-001:

Repository Pattern mandatory.
```

# 7. Risk Analysis

Ogni modifica riceve un livello rischio.

---

## LOW

```text id="x4m8p2"
Cambio funzione isolata

Aggiunta test

Refactoring locale
```

---

## MEDIUM

```text id="p9n3m7"
Nuovo modulo

Nuova libreria

Cambio API interna
```

---

## HIGH

```text id="q6m2x8"
Database migration

Cambio architettura

Security changes

Breaking API
```

# 9. Root Cause Analysis

Quando un test fallisce:

Non correggere subito.

Prima:

```text id="k3m7x9"

Error

↓

Classification

↓

Root Cause

↓

Solution
```

---

Esempio:

Errore:

```text id="z8n4m2"
NullPointerException
```

Risposta sbagliata:

```text id="s5q9p3"
Aggiungi if null
```

---

Risposta corretta:

```text id="m2x7k8"
User object può essere creato senza email.

Modificare validation layer.
```

# Principio finale

La differenza tra un autocomplete evoluto e un software engineer è il processo decisionale.

Scrivere codice è solo una parte.

Un buon agente deve prima chiedersi:

```text id="z4m8q3"
"Qual è il problema reale?"

"Quali soluzioni esistono?"

"Quale scelta mantiene il sistema sano tra un anno?"
```

Solo dopo deve scrivere codice.
