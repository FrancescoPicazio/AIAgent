# AI Software Engineer Agent - Parte 49

# Database interno dell'agente: stato persistente, schema dati, eventi, task history e gestione multi-progetto

## Obiettivo

Definire il database interno che permette all'AI Software Engineer di mantenere stato, memoria e conoscenza operativa tra una sessione e l'altra.

Il database interno non sostituisce:

* Vector Database → conoscenza semantica;
* Git → storia del codice;
* File `.md` → documentazione leggibile.

Serve per gestire:

* stato workflow;
* utenti;
* task;
* agent execution;
* approvazioni;
* configurazioni;
* metriche;
* eventi.

---

# 1. Scelta Database

Per un sistema locale:

## SQLite

Consigliato inizialmente.

Vantaggi:

* zero configurazione;
* file locale;
* ottimo per sviluppo.

Per versione avanzata.

Vantaggi:

* multiutente;
* concorrenza;
