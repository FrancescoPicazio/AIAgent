# AI Software Engineer Agent - Parte 1

# Fondazioni del progetto e principi architetturali

## Obiettivo

Realizzare un agente AI locale in grado di comportarsi come un software engineer autonomo di livello senior.

L'agente deve essere in grado di:

* analizzare requisiti descritti in linguaggio naturale;
* comprendere un progetto software esistente;
* progettare nuove funzionalità;
* suddividere il lavoro in task;
* modificare il codice;
* eseguire test;
* correggere automaticamente gli errori;
* mantenere memoria del progetto;
* documentare le modifiche;
* chiedere approvazione umana nei punti critici.

L'obiettivo non è creare un semplice assistente di coding, ma un sistema capace di seguire un ciclo completo di sviluppo software.

---

# Principi fondamentali

## 1. Human in the loop

L'utente mantiene sempre l'approvazione finale.

L'agente può:

* analizzare;
* proporre soluzioni;
* implementare modifiche approvate;
* eseguire test;
* proporre miglioramenti.

L'agente non può autonomamente:

* cambiare architettura principale;
* eliminare funzionalità;
* modificare elementi critici senza approvazione;
* effettuare merge definitivi senza conferma.

---

## 2. Local first

L'intero sistema deve funzionare localmente.

Vincoli:

* nessun codice deve uscire dalla macchina locale;
* nessuna dipendenza obbligatoria da API cloud;
* modelli eseguiti tramite Ollama;
* memoria e database locali.

Obiettivo:

Garantire privacy, controllo e possibilità di personalizzazione completa.

# Stack tecnologico

## LLM Runtime

### Ollama

Responsabilità:

* gestione modelli locali;
* inferenza;
* gestione quantizzazione;
* esposizione API locali.

Modelli previsti:

* modello leggero per orchestrazione;
* modello coding per generazione codice;
* eventuali modelli specializzati per review e reasoning.

---

## Framework Agentico

### LangChain

Utilizzato per:

* gestione tool;
* prompt;
* retrieval;
* integrazione modelli;
* gestione memoria.

---

### LangGraph

Utilizzato come orchestratore principale.

Responsabilità:

* definizione workflow;
* gestione stato;
* transizioni tra agenti;
* gestione loop di correzione;
* supervisione task.

---

## Monitoring e valutazione

### LangSmith

Utilizzato come sistema di osservabilità.

Obiettivi:

* tracciare chiamate ai modelli;
* analizzare prompt e risposte;
* misurare qualità degli agenti;
* individuare errori;
* valutare regressioni.

Anche se il sistema gira localmente, LangSmith sarà utilizzato come criterio di valutazione del comportamento dell'agente.

# Architettura generale

Il sistema sarà composto da agenti specializzati.

## Architect Agent

Responsabilità:

* analizzare requisiti;
* proporre soluzioni architetturali;
* valutare impatto delle modifiche;
* creare decisioni tecniche.

Output:

* documento architetturale;
* piano tecnico;
* rischi identificati.

---

## Planner Agent

Responsabilità:

Trasformare una richiesta in task atomici.

Esempio:

Input:

"Aggiungere autenticazione JWT"

Output:

```
Task 1:
Creare modello utente

Task 2:
Implementare repository utenti

Task 3:
Creare servizio JWT

Task 4:
Aggiungere middleware

Task 5:
Creare test
```

Ogni task deve contenere:

* descrizione;
* file coinvolti;
* funzioni impattate;
* livello rischio;
* dipendenze.

---

## Coding Agent

Responsabilità:

* modificare codice;
* creare nuovi file;
* rispettare regole del progetto;
* produrre patch applicabili.

Il coding agent non decide l'architettura.

---

## Reviewer Agent

Responsabilità:

* analizzare qualità del codice;
* verificare rispetto delle regole;
* individuare problemi;
* proporre miglioramenti.

---

## Testing Agent

Responsabilità:

* eseguire pipeline di validazione;
* analizzare errori;
* attivare loop di correzione.

Un task è completato solo quando:

```
Codice
 |
 ↓
Lint
 |
 ↓
Test
 |
 ↓
Review
 |
 ↓
Aggiornamento memoria
 |
 ↓
Completato
```

---

# Filosofia del sistema

L'agente deve ragionare come un membro di un team software.

Prima di modificare:

1. Comprende il problema.
2. Analizza il progetto.
3. Valuta impatto.
4. Propone una soluzione.
5. Attende approvazione quando necessario.
6. Implementa.
7. Verifica.
8. Aggiorna la conoscenza del progetto.

---

# Obiettivo finale

Creare un ambiente di sviluppo dove l'utente possa interagire tramite chat:

"Esegui questa modifica"

e l'agente sia in grado di:

* capire il progetto;
* pianificare il lavoro;
* modificare il codice;
* validare il risultato;
* mantenere coerenza nel tempo.

Il sistema deve evolvere da semplice generatore di codice a vero agente di sviluppo software autonomo.
