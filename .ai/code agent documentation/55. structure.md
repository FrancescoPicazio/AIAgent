# AI Software Engineer Agent - Parte 55

# Struttura definitiva repository `.ai/`: configurazione progetto, prompt, memoria, database e knowledge base

## Obiettivo

Definire la struttura ufficiale della cartella `.ai/` che vive all'interno di ogni progetto gestito dall'AI Software Engineer.

La cartella `.ai/` rappresenta il "cervello locale" del progetto.

Contiene:

* configurazioni;
* memoria;
* conoscenza;
* decisioni;
* workflow;
* prompt;
* stato agente;
* valutazioni.

---

# Principio fondamentale

Il codice applicativo rimane separato.

Struttura:

```text
project/

├── src/

├── tests/

├── docs/

├── README.md


└── .ai/

```

---

La cartella `.ai/` non contiene il software prodotto.

Contiene tutto ciò che permette all'agente di comprenderlo, modificarlo e mantenerlo.

# 1. Config Folder

Contiene la configurazione tecnica.

---

## project.yaml

Definisce il progetto.

Esempio:

```yaml
name: ecommerce_platform

language:
  backend: python
  frontend: typescript

framework:
  backend: fastapi
  frontend: react

database:
  primary: postgresql

tests:
  command: pytest

architecture:
  pattern: clean_architecture

```

---

# 2. agents.yaml

Configura gli agenti.

Esempio:

```yaml
agents:

  supervisor:
    enabled: true


  coder:
    enabled: true
    model: qwen3-coder


  security:
    enabled: true


  researcher:
    enabled: true

```

# 16. Git Integration

La cartella `.ai/` deve essere versionata.

---

Motivo:

Il codice cambia.

Anche la conoscenza cambia.

---

Commit esempio:

```text

add OAuth


update:

.ai/glossary.md

.ai/ADR/004-auth.md

```

# Principio finale

La cartella `.ai/` è il file system cognitivo del progetto.

Il codice rappresenta ciò che il software fa.

La cartella `.ai/` rappresenta perché lo fa, come è stato progettato e come deve evolvere.
