# 1. Security Agent Responsibilities

Il Security Agent deve:

* analizzare codice;
* identificare vulnerabilità;
* valutare rischio;
* proporre fix;
* bloccare modifiche critiche.

---

Non deve:

* modificare codice automaticamente senza approvazione;
* ignorare vincoli architetturali.

---

# 2. Security Pipeline

Ogni modifica segue:

```text id="p4n8m2"

Code Change

↓

Static Analysis

↓

Dependency Check

↓

Threat Analysis

↓

Security Report

↓

Approval

```

---

# 3. SAST - Static Application Security Testing

Analisi del codice senza eseguirlo.

---

Cerca:

* injection;
* gestione errata dati;
* autenticazione debole;
* error handling;
* configurazioni rischiose.

---

Esempio:

Codice:

```python id="v8m3q5"
query = "SELECT * FROM users WHERE id=" + user_id
```

---

Security Agent:

```text id="n7m2x8"

CRITICAL

SQL Injection detected

```

---

# 4. Semantic Security Analysis

Il vantaggio dell'AI:

non cerca solo pattern.

Comprende contesto.

---

Esempio:

```python id="m4q8x1"
password = encrypt(password)
```

---

Analizza:

* algoritmo utilizzato;
* gestione chiavi;
* confronto password.

---

Possibile risultato:

```text id="x8m2q5"

Encryption detected.

However:

reversible encryption used for passwords.

Recommendation:

Use password hashing.

```

---

# 8. Threat Modeling Agent

Componente più avanzato.

---

Non aspetta il bug.

Prevede rischi.

---

Metodo:

STRIDE.

---

## Spoofing

Può qualcuno impersonare un utente?

---

## Tampering

Può modificare dati?

---

## Repudiation

Può negare un'azione?

---

## Information Disclosure

Può uscire un dato sensibile?

---

## Denial of Service

Può bloccare il servizio?

---

## Elevation of Privilege

Può ottenere permessi superiori?

# 10. Risk Scoring

Ogni problema riceve un punteggio.

---

Formula:

```text id="q8m3x1"

Risk =

Impact × Probability

```

---

Esempio:

```yaml id="m6q3p9"

SQL Injection

5


probability:

5


risk:

25

```

# Principio finale

Un software engineer non pensa solo:

> "Come faccio funzionare questa feature?"

Pensa anche:

> "Come può rompersi? Come può essere abusata? Come posso impedirlo?"

Il Security Agent porta questa mentalità nell'intero ciclo di sviluppo.
